## Honest framing

The headline instruction was "full run, fill every empty field, fact check everything." The research half did that at scale (55 areas + 5 major operator groups researched in one wave, ~5,700 field-level proposals produced). The application half applied only 7 edits because mechanically applying 5,700 field-level edits across 55+ TypeScript files in a single session is outside the envelope for one agent pass.

The proposals on disk are **durable structured source material** — they don't decay, and a follow-up apply-only session can consume them linearly to land the full enrichment. This run did the expensive half. The cheap half is mechanical.

The alternative — dispatching agents to both research and edit source — risks 55 concurrent file edits colliding, is harder to review, and dilutes the "human-review the research, agent writes the file" discipline that prior runs have used successfully.

**This is how Flatbrowser data gets to the 266-project × 40-field-per-project depth without losing epistemic quality.** One run does the research; the next run applies it. Total cost per round is ~1M-2M tokens; total quality improvement per round is meaningful and inspectable.
